# EduOofa Counselor CRM Design

## 1. CRM Purpose

The counselor CRM helps counselors convert student interest into successful applications and mentorship outcomes.

## 2. Counselor Dashboard

Key widgets:

- Assigned leads.
- Follow-ups due today.
- Upcoming mentorship sessions.
- Open applications.
- Students without recent contact.
- Conversion metrics.
- Revenue influenced, if applicable.

## 3. Lead Pipeline

Pipeline stages:

- New Lead.
- Profile Incomplete.
- Counseling Pending.
- Shortlist Created.
- Application Started.
- Application Submitted.
- Partner Follow-Up.
- Converted.
- Lost.

Loss reasons:

- Budget mismatch.
- Eligibility mismatch.
- No response.
- Joined another institution.
- Not interested.
- Duplicate lead.

## 4. Student CRM Profile

Sections:

- Student identity and contact.
- Academic background.
- Entrance exams and scores.
- Preferences.
- Budget.
- Location preference.
- Saved colleges.
- Recommended colleges.
- Applications.
- Bookings.
- Notes.
- Call logs.
- Tasks.
- Documents, later phase.

## 5. Counselor Actions

- Add note.
- Create task.
- Log call.
- Schedule follow-up.
- Recommend colleges.
- Update pipeline stage.
- Assign application priority.
- Book mentorship session.
- Add session outcome.

## 6. Task Management

Task fields:

- Student.
- Counselor.
- Type.
- Due date.
- Priority.
- Status.
- Description.
- Completion notes.

Task types:

- Call.
- WhatsApp.
- Email.
- Document Reminder.
- Application Follow-Up.
- Payment Follow-Up.
- Session Preparation.

## 7. Assignment Rules

Initial assignment options:

- Manual admin assignment.
- Round-robin by counselor availability.
- Course specialization match.
- Region/language match.
- Workload-aware assignment.

MVP should start with manual assignment plus optional round-robin.

## 8. CRM Permissions

Counselors can access assigned students only. Admins and super admins can access all CRM data. All notes and call logs are immutable after a configured edit window unless admin override is granted.
